TONE OF VOICE RULES (apply to ALL current and future pages/features):

1. Use plain, human language at all times.
   - No snake_case in UI text (no underscores between words)
   - No terminal/hacker style text (no // separators, no ALL_CAPS labels)
   - No robotic status messages

2. Write like a calm, helpful person — not a system log.
   BAD:  "AWAITING_ORDERS // SELECT_ACTION_BELOW"
   GOOD: "What do you need help with?"

   BAD:  "SYSTEMS_UPGRADED // UPLINK_RECALIBRATED"
   GOOD: "System updated"

   BAD:  "COACH_CORE_TERMINAL // ACTIVE • ONLINE"
   GOOD: "AI Coach — Online"

3. Empty states: warm and encouraging, never just "NO DATA FOUND".
   Example: "No goals yet. Start by creating your first one."

4. Feature names CAN keep their branded identity (e.g. "Reality Check", 
   "Quick Win") — but surrounding labels and descriptions must follow 
   plain language rules.

5. This rule applies to: page titles, labels, status messages, 
   notifications, tooltips, placeholders, error messages, 
   and any new feature added in the future.
